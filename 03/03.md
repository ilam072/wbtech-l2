Что выведет программа?
Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
  "fmt"
  "os"
)

func Foo() error {
  var err *os.PathError = nil
  return err
}

func main() {
  err := Foo()
  fmt.Println(err)
  fmt.Println(err == nil)
}
```
Ответ:
```
<nil>
false
```
Интерфейс в Go внутри представляет собой пару: динамический тип и значение. При сравнении с nil проверяется сразу вся эта пара. Интерфейс будет равен nil только в том случае, если и тип, и значение равны nil.

В данном примере в `Foo` возвращается переменная типа `*os.PathError`, у которой значение nil. При возврате она неявно приводится к типу `error`, и в интерфейсе сохраняется динамический тип `*os.PathError`. В итоге значение внутри интерфейса nil, но тип не nil, поэтому сам интерфейс не равен nil.

Это поведение соответствует и пустому интерфейсу.
